### Список операций ОПС

В ОПС записываются следующие типы операций:

1.  **Бинарные арифметические операции:**
    *   `+` — сложение;
    *   `-` — вычитание;
    *   `*` — умножение;
    *   `/` — деление.
2.  **Унарная арифметическая операция:**
    *   `–'` — унарный минус (инверсия знака). Генерируется в ОПС после операнда.
3.  **Операции сравнения:**
    *   `==`, `!=`, `<`, `>`, `<=`, `>=` — логические операции, результат которых помещается в стек.
4.  **Операция присваивания:**
    *   `:=` — требует двух операндов: адреса переменной и значения.
5.  **Операции ввода-вывода:**
    *   `r` (read) — операция чтения в переменную. Операнд должен быть ссылкой на переменную.
    *   `w` (write) — операция вывода значения выражения.
6.  **Операция работы с массивами:**
    *   `i` — индексирование в одномерном массиве. Вычисляет адрес элемента: `базовый_адрес + индекс`.
7.  **Операции управления (переходы):**
    *   `j` — безусловный переход по адресу.
    *   `jf` — переход по условию "ложь". Требует два операнда: логическое значение и адрес перехода.

### Формат ОПС

ОПС представляется в виде линейной последовательности элементов. 

**Структура одного элемента ОПС состоит из двух частей:**
1.  **Тип элемента:**
    *   `a` — ссылка на переменную (указывает на таблицу скалярных переменных или описателей массивов);
    *   `k` — ссылка на константу (указывает на таблицу констант);
    *   `O` — операция (+, :=, jf и т.д.);
    *   `m` — метка (номер элемента в массиве ОПС, используется для переходов).
2.  **Значение элемента:**
    *   Для операндов (`a`, `k`) — это индекс в соответствующей таблице.
    *   Для операций (`O`) — это уникальный код конкретной операции.
    *   Для меток (`m`) — это индекс элемента ОПС, на который будет совершен переход.

**Модель вычислений (Интерпретация):**
Вычисление ОПС выполняется интерпретатором с использованием **магазина (стека)**. 
*   Если элемент ОПС — **операнд** (ссылка на переменную или константу), то в магазин заносится его вид и ссылка.
*   Если элемент ОПС — **арифметическая/логическая операция**, из магазина извлекаются операнды, выполняется действие, и результат записывается обратно в магазин.
*   Для **операции присваивания (`:=`)** первый операнд должен быть ссылкой на адрес, второй — значением. Значение записывается по ссылке, в магазин ничего не возвращается.
*   Для **операций перехода (`j`, `jf`)** значение счетчика команд интерпретатора заменяется на значение метки, если условие выполнено.
