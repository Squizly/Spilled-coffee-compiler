### Список операций ОПС

В ОПС записываются следующие типы операций:

1.  **Бинарные арифметические операции:**
    *   `+` — сложение;
    *   `-` — вычитание;
    *   `*` — умножение;
    *   `/` — деление.
    
    Все четыре операции — **полиморфные**: тип результата определяется в момент исполнения по типам операндов в стеке (см. правила преобразований ниже).

2.  **Унарная арифметическая операция:**
    *   `–'` — унарный минус (инверсия знака). Генерируется в ОПС после операнда. Тип результата совпадает с типом операнда.

3.  **Операции сравнения:**
    *   `==`, `!=`, `<`, `>`, `<=`, `>=` — логические операции, результат которых (целое 0 или 1) помещается в стек. Поддерживают смешанные типы операндов с автоматическим приведением `int → float`.

4.  **Операция присваивания:**
    *   `=` — требует двух операндов: адреса переменной и значения. Поддерживает неявные преобразования `int ↔ float` в соответствии с типом переменной-приёмника.

5.  **Операции ввода-вывода:**
    *   `r` (read) — операция чтения в переменную. Операнд должен быть ссылкой на переменную; вводимое значение интерпретируется согласно типу переменной (`int` или `float`).
    *   `w` (write) — операция вывода значения выражения. Формат вывода зависит от типа значения на вершине стека.

6.  **Операция работы с массивами:**
    *   `i` — индексирование в одномерном массиве. Вычисляет адрес элемента: `базовый_адрес + индекс`.

7.  **Операции управления (переходы):**
    *   `j` — безусловный переход по адресу.
    *   `jf` — переход по условию "ложь". Требует два операнда: логическое значение и адрес перехода.

8.  **Стандартные функции (унарные):**
    *   `sqrt` — извлечение квадратного корня. Берёт верхний операнд из магазина; если операнд `int`, он сначала приводится к `float`. Кладёт результат типа **`float`**. Ошибка времени выполнения, если аргумент `< 0`.
    *   `exp` — экспонента e^x. Берёт верхний операнд (с приведением `int → float` при необходимости), кладёт результат типа **`float`**. Ошибка при переполнении.
    *   `log` — натуральный логарифм. Берёт верхний операнд (с приведением `int → float` при необходимости), кладёт результат типа **`float`**. Ошибка времени выполнения, если аргумент `≤ 0`.

### Формат ОПС

ОПС представляется в виде линейной последовательности элементов.

**Структура одного элемента ОПС состоит из двух частей:**
1.  **Тип элемента:**
    *   `a` — ссылка на переменную (указывает на таблицу скалярных переменных или описателей массивов; в этой таблице хранится тип переменной — `int` или `float`);
    *   `k` — ссылка на константу (указывает на таблицу констант; в этой таблице хранится тип литерала — `int` или `float`);
    *   `O` — операция (+, :=, jf, sqrt, exp, log и т.д.);
    *   `m` — метка (номер элемента в массиве ОПС, используется для переходов).
2.  **Значение элемента:**
    *   Для операндов (`a`, `k`) — это индекс в соответствующей таблице.
    *   Для операций (`O`) — это уникальный код конкретной операции.
    *   Для меток (`m`) — это индекс элемента ОПС, на который будет совершен переход.

### Модель вычислений (Интерпретация)

Вычисление ОПС выполняется интерпретатором с использованием **магазина (стека)**. Каждый элемент стека хранит **значение и его тип** (`int` или `float`).

*   Если элемент ОПС — **операнд** (ссылка на переменную или константу), то в магазин заносится его значение и тип (тип берётся из соответствующей таблицы).
*   Если элемент ОПС — **арифметическая/логическая операция**, из магазина извлекаются операнды, выполняется приведение типов согласно правилам ниже, выполняется действие, и результат с вычисленным типом записывается обратно в магазин.
*   Для **операции присваивания (`=`)** первый операнд должен быть ссылкой на адрес, второй — значением. Перед записью значение приводится к типу переменной-приёмника. В магазин ничего не возвращается.
*   Для **операций перехода (`j`, `jf`)** значение счетчика команд интерпретатора заменяется на значение метки, если условие выполнено.
*   Для **стандартных функций (`sqrt`, `exp`, `log`)** из магазина извлекается один операнд, при необходимости приводится к `float`, выполняется вычисление, результат типа `float` помещается обратно в магазин.

### Правила преобразования типов во время исполнения

| Операция | Типы операндов | Действие | Тип результата |
| :--- | :--- | :--- | :--- |
| `+`, `-`, `*` | int, int | прямое целочисленное | int |
| `+`, `-`, `*` | int, float (в любом порядке) | int → float, затем вещественное | float |
| `+`, `-`, `*` | float, float | прямое вещественное | float |
| `/` | int, int | **целочисленное** деление; ошибка при делителе 0 | int |
| `/` | int, float (в любом порядке) | int → float, затем вещественное; ошибка при делителе 0.0 | float |
| `/` | float, float | вещественное; ошибка при делителе 0.0 | float |
| `–'` | int | инверсия знака | int |
| `–'` | float | инверсия знака | float |
| `==`, `!=`, `<`, `>`, `<=`, `>=` | смешанные | int → float, затем сравнение | int (0 или 1) |
| `==`, `!=`, `<`, `>`, `<=`, `>=` | одинаковые | прямое сравнение | int (0 или 1) |
| `=` | приёмник int, значение int | прямое присваивание | — |
| `=` | приёмник float, значение int | расширение int → float | — |
| `=` | приёмник int, значение float | усечение float → int (отбрасывание дробной части) | — |
| `=` | приёмник float, значение float | прямое присваивание | — |
| `sqrt`, `exp`, `log` | int | int → float, затем функция | float |
| `sqrt`, `exp`, `log` | float | прямое применение функции | float |

### Возможные ошибки времени выполнения

* Деление на `0` (для `int`) или `0.0` (для `float`).
* `sqrt(x)` при `x < 0`.
* `log(x)` при `x ≤ 0`.
* `exp(x)` при переполнении вещественного диапазона.
* Выход индекса массива за границы.
* Чтение неинициализированной переменной (опционально).

При любой ошибке времени выполнения интерпретатор сообщает её вид и привязку к исходному тексту (номер строки и позиция символа), которые сохраняются в ОПС вместе с операцией на этапе генерации.
