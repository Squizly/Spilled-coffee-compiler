# Семантические действия для генерации ОПС

Анализатор LL(1) и генератор ОПС реализованы единой совмещённой таблицей. В каждой клетке: **правая часть** правила (помещаемая в магазин LL(1)-анализатора в обратном порядке) и **строка семантических действий** ровно той же длины (помещаемая в дополнительный магазин). Действия извлекаются и выполняются синхронно с символами правой части.

## Семантические действия

- **1, 2, 3, 4, 5** — программы для генерации меток и переходов (`jf` — переход по лжи, `j` — безусловный переход) для `if` и `while`.
- **6, 7** — установка «текущего типа» = `int` / `float` для последующих описаний.
- **8, 9** — занесение `id` в таблицу переменных как скаляра / массива.
- **a** — запись в ОПС адреса (имени) переменной.
- **k** — запись в ОПС константы.
- **`+`, `–`, `*`, `/`** — бинарные арифметические операции.
- **`@`** — унарный минус.
- **`i`** — операция индексации массива.
- **`=`** — операция присваивания (в составе колонки `==/<...` означает оператор сравнения «равно»).
- **`r`, `w`** — операции чтения (`input`) и вывода (`print`).
- **`==`, `≠`, `<`, `>`, `≤`, `≥`** — операции сравнения.
- **`s`, `e`, `l`** — вызовы функций `sqrt`, `exp`, `log`.
- **`□`** — пустое действие.

### Программа 1 — переход по лжи
В ОПС записывается элемент-метка (пустое место) и команда `jf`. Адрес пустого места заносится в семантический стек меток `MM`.

### Программа 2 — безусловный переход для else
В ОПС записывается элемент-метка и команда `j`. Из `MM` извлекается адрес от программы **1**, в это место записывается адрес следующей за `j` команды. Адрес нового пустого места заносится в `MM`.

### Программа 3 — завершение if
В ОПС ничего не записывается. Из `MM` извлекается последний адрес (от **1**, если `else` отсутствует, или от **2**, если есть) и в него записывается значение текущего счётчика ОПС.

### Программа 4 — фиксация начала цикла
В ОПС ничего не записывается. В `MM` заносится текущий адрес (точка возврата для повторной проверки условия).

### Программа 5 — завершение цикла
В ОПС записывается «адрес начала» (сохранённый программой **4**) и команда `j`. Из `MM` извлекается адрес перехода по лжи (от программы **1**), в это место записывается адрес текущей позиции.

### Программы 6–9 — описания переменных
- **6** — установить текущий тип = `int`.
- **7** — установить текущий тип = `float`.
- **8** — занести `id` в таблицу переменных как скаляр с текущим типом.
- **9** — переклассифицировать только что занесённый `id` как массив с размером, заданным предыдущим литералом `num_int`.

## Правила работы с типами (int / float)

В таблице переменных для каждого имени хранится тип, установленный программами **6–9**. В магазине интерпретатора каждый элемент несёт пометку типа значения; тип результата операции вычисляется в момент исполнения.

**Бинарные арифметические `+ – * /`.** `int ⊗ int → int`; деление `int/int` целочисленное, деление на `0` — ошибка. Если хотя бы один операнд `float` — второй приводится к `float`, результат `float`; деление на `0.0` — ошибка.

**Сравнения.** При смешанных типах `int` приводится к `float`, далее вещественное сравнение. Результат — `int` (`0` или `1`).

**Присваивание `=`.** Совпадающие типы — без преобразования. `float = int` — расширение. `int = float` — усечение дробной части.

**Стандартные функции `sqrt`, `exp`, `log`.** Один аргумент, всегда возвращают `float` (с неявным `int → float` при необходимости). Время выполнения: `sqrt` ошибка при `x < 0`, `log` при `x ≤ 0`, `exp` при переполнении.

**Унарный минус `@`.** Тип результата совпадает с типом операнда.
