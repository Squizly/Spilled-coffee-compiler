# Семантические действия для генерации ОПС

## Обозначения

- `□` – пустое действие
- `v` – записать переменную в ОПС (с типом из таблицы переменных)
- `k` – записать константу в ОПС (с типом литерала: int / float)
- `+, -, *, /, :=, <, >, ==, !=, <=, >=` – записать операцию в ОПС
- `–'` – записать унарный минус (отличается от бинарного `-`)
- `i` – индексация массива
- `r` – оператор чтения (input)
- `w` – оператор вывода (print)
- `j` – безусловный переход
- `jf` – переход если false
- `m1`, `m2`, `m3` – создать метку (запомнить адрес перехода)
- `sqrt`, `exp`, `log` – записать вызов стандартной функции (унарная операция)

## Правила работы с типами (int / float)

В таблице переменных для каждого имени хранится тип (`int` или `float`), установленный при объявлении. Для каждого элемента стека интерпретатора во время выполнения также хранится тип значения. Тип результата каждой операции вычисляется интерпретатором в момент её исполнения.

### Преобразования при бинарных арифметических операциях (`+`, `-`, `*`, `/`)

1. Если оба операнда `int` — результат `int`. Деление `int / int` выполняется как **целочисленное** (отбрасывание дробной части); деление на `0` — ошибка времени выполнения.
2. Если хотя бы один операнд `float` — второй неявно приводится к `float`, результат `float`. Деление `float / float` выполняется как обычное вещественное; деление на `0.0` — ошибка времени выполнения.

### Преобразования при сравнениях (`==`, `!=`, `<`, `>`, `<=`, `>=`)

При смешанных типах (`int` и `float`) операнд типа `int` неявно приводится к `float`, после чего применяется вещественное сравнение. Результат сравнения логический (трактуется как `int` 0/1 в стеке).

### Преобразования при присваивании (`:=`)

1. `int := int`, `float := float` — без преобразования.
2. `float := int` — целое значение неявно расширяется до вещественного.
3. `int := float` — вещественное значение неявно усекается до целого (отбрасывание дробной части), с предупреждением о возможной потере точности (опционально).

### Стандартные функции (`sqrt`, `exp`, `log`)

Все три функции принимают один аргумент и **всегда возвращают `float`**, даже если аргумент имел тип `int` (в этом случае происходит неявное преобразование `int → float` перед вычислением).

Дополнительные ограничения времени выполнения:
- `sqrt(x)` — ошибка, если `x < 0`.
- `log(x)` — ошибка, если `x ≤ 0`.
- `exp(x)` — ошибка при переполнении (если результат выходит за допустимый диапазон).

### Унарный минус (`–'`)

Тип результата совпадает с типом операнда: `–' int → int`, `–' float → float`.
