# Лексический Анализатор

Данный документ описывает логику работы сканера, который преобразует исходный текст программы в поток токенов (лексем) для последующего синтаксического анализа.

## 1. Реестр Лексем

Каждой распознанной лексеме присваивается уникальный числовой код.  
Код `0` зарезервирован для маркера конца файла ($\perp$), код `-1` — для ошибок.

### Служебные
| Имя лексемы     | Код  | Описание                  |
|-----------------|------|---------------------------|
| `T_EOF`         | 0    | Конец файла               |
| `T_ERROR`       | -1   | Ошибочный символ / неверный формат |

### Ключевые слова и типы
| Имя лексемы | Код | Описание |
| :--- | :--- | :--- |
| `T_IF`, `T_ELSE` | 4, 5 | Условные конструкции |
| `T_WHILE` | 6 | Цикл |
| `T_INPUT`, `T_PRINT`| 7, 8 | Ввод/вывод |
| `T_INT`, `T_FLOAT` | 9, 10 | Типы данных |

### Литералы и идентификаторы
| Имя лексемы | Код | Описание |
| :--- | :--- | :--- |
| `T_ID` | 1 | Идентификатор (латинские буквы, цифры и `_`) |
| `T_NUM_INT` | 2 | Целое число |
| `T_NUM_FLOAT`| 3 | Вещественное число (через **точку**) |

### Операторы и сравнения
| Имя лексемы | Код | Символы |
| :--- | :--- | :--- |
| `T_PLUS`, `T_MINUS` | 11, 12 | `+`, `-` |
| `T_MUL`, `T_DIV` | 13, 14 | `*`, `/` |
| `T_ASSIGN` | 15 | `=` |
| `T_EQ`, `T_NE` | 16, 17 | `==`, `!=` |
| `T_LT`, `T_GT` | 18, 19 | `<`, `>` |
| `T_LE`, `T_GE` | 20, 21 | `<=`, `>=` |

### Разделители
| Имя лексемы | Код | Символы |
| :--- | :--- | :--- |
| `T_LPAR`, `T_RPAR` | 22, 23 | `(`, `)` |
| `T_LBRACKET`, `T_RBRACKET`| 24, 25 | `[`, `]` |
| `T_LBRACE`, `T_RBRACE` | 26, 27 | `{`, `}` |
| `T_COMMA`, `T_SEMICOLON` | 28, 29 | `,`, `;` |

## 2. Таблица переходов конечного автомата

### Состояния автомата
| Состояние | Описание |
| :--- | :--- |
| **S0** | Начальное состояние: ожидание новой лексемы |
| **S1** | Идентификатор: накопление букв, цифр и `_` |
| **S2** | Целое число: накопление цифр |
| **S3** | После точки (ожидание первой цифры дробной части)|
| **S4** | Дробная часть числа |
| **S5** | После символов `=`, `<`, `>` (потенциально составной оператор) |
| **S6** | После `!` (ожидается `=`) |

### Семантические действия
| Код | Действие |
| :--- | :--- |
| **0** | Очистить буфер и записать текущий символ |
| **1** | Добавить текущий символ в буфер |
| **2** | Пропустить пробельный символ (ничего не делать) |
| **3** | Выдать `T_ID` или ключевое слово + **откат** |
| **4** | Выдать `T_NUM_INT` + **откат** |
| **5** | Выдать `T_NUM_FLOAT` + **откат** |
| **6** | Выдать одиночный оператор или разделитель (без отката) |
| **7** | Выдать составной оператор (`==`, `<=`, `>=`, `!=`) |
| **8** | Выдать одиночный `=`, `<`, `>` + **откат** |
| **9** | Сформировать ошибку `T_ERROR` (недопустимый символ или неверный формат) |

### Матрица переходов

*Обозначения: **S** — финальное псевдосостояние (выдача лексемы и возврат в S0), **M** — состояние ошибки.*

| Текущее | Буква / `_` | Цифра | `.` | `, ;` | `=` | `<` `>` | `!` | `+ - * /` | `()[]{}` | Пробел | Прочее |
| :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **S0** | S1/0 | S2/0 | M/9 | **S/6** | S5/0 | S5/0 | S6/0 | **S/6** | **S/6** | S0/2 | M/9 |
| **S1** | S1/1 | S1/1 | **S/3** | **S/3** | **S/3** | **S/3** | **S/3** | **S/3** | **S/3** | **S/3** | M/9 |
| **S2** | M/9 | S2/1 | S3/1 | **S/4** | **S/4** | **S/4** | **S/4** | **S/4** | **S/4** | **S/4** | M/9 |
| **S3** | M/9 | S4/1 | M/9 | M/9 | M/9 | M/9 | M/9 | M/9 | M/9 | M/9 | M/9 |
| **S4** | M/9 | S4/1 | M/9 | **S/5** | **S/5** | **S/5** | **S/5** | **S/5** | **S/5** | **S/5** | M/9 |
| **S5** | **S/8** | **S/8** | **S/8** | **S/8** | **S/7** | **S/8** | **S/8** | **S/8** | **S/8** | **S/8** | M/9 |
| **S6** | M/9 | M/9 | M/9 | M/9 | **S/7** | M/9 | M/9 | M/9 | M/9 | M/9 | M/9 |

> **Примечание:** Действия, выделенные **жирным**, являются финальными (лексема распознана).

### Пояснения к логике работы матрицы:
* Неполные вещественные литералы (`.5`, `5.`) намеренно считаются ошибкой
* Идентификатор не может содержать точку (`ab.cd` -> `T_ID(ab)` + `T_ERROR` на точке)
* После цифр буквы или другие недопустимые символы (`123abc`, `5.23@`) приводят к `T_ERROR`
